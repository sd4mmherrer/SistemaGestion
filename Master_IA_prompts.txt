CONTEXTO DEL PROYECTO

El sistema debe demostrar separación de responsabilidades entre:
1. Presentación
2. Lógica de Negocio
3. Acceso a Datos
4. Entidades

Además, el proyecto debe incluir operaciones CRUD completas, validaciones, manejo de errores, persistencia en archivos .txt y buenas prácticas de programación.
También debo documentar el uso de IA, por lo que necesito que todo lo que generes sea comprensible, explicable y adaptable, no algo excesivamente complejo ni demasiado profesional.

OBJETIVO
Quiero que generes un proyecto funcional, limpio y bien estructurado, con un nivel intermedio: que se vea bien hecho, pero que yo pueda entenderlo, explicarlo y defenderlo fácilmente.

IMPORTANTE
NO quiero una solución con base de datos.
NO quiero mezclar capas.
NO quiero código innecesariamente avanzado.
NO quiero librerías externas si no son necesarias.
Quiero que uses únicamente Java estándar y archivos .txt para la persistencia.

TEMA DEL SISTEMA

REQUERIMIENTOS OBLIGATORIOS
El sistema debe permitir:
- Agregar registro

PERSISTENCIA

id|campo1|campo2|campo3

ARQUITECTURA OBLIGATORIA
Organiza el proyecto estrictamente en capas, por ejemplo:

/entidades
/logica
/accesodatos
/presentacion

No mezcles responsabilidades entre capas.
La capa de presentación no debe acceder directamente a archivos.
La capa de acceso a datos no debe contener lógica de interacción con el usuario.
La lógica de negocio debe coordinar validaciones y reglas.
Las entidades solo deben representar datos.

VALIDACIONES OBLIGATORIAS
Incluye:
- Validación de campos vacíos
- Validación de tipos de datos
- Manejo de errores con try-catch
- Uso correcto de try-with-resources en manejo de archivos
